# PassVault — 最终设计文档

> **版本**: v2.0  
> **日期**: 2026-06-06  
> **技术栈**: Electron + Vanilla HTML/CSS/JS  
> **目标平台**: Windows

---

## 1. 项目概述

PassVault 是一款轻量级桌面密码管理器。

- 本地 AES-256-GCM 加密存储，WebDAV/本地文件夹云端同步
- 主密码 + 恢复密钥双入口解密
- 条目标签化多 Vault 支持
- 自动锁屏、回收站、剪贴板管理

### 核心原则

| 原则 | 说明 |
|------|------|
| 零信任 | 加密在本地完成，云端只存密文 |
| 双入口 | 主密码 (日常) / 恢复密钥 (应急) 两种解密方式 |
| 整文件加密 | 单一 `.pvault` 文件，整文件级加密 |
| 内存解密 | 解锁后全部解密到内存，锁屏时擦除 |
| 无框架 | 纯 HTML/CSS/JS，零额外依赖 |

| 层 | 技术 | 说明 |
|----|------|------|
| 桌面壳 | Electron 31+ | Node.js 已可用 |

## 4. 加密方案

### 4.1 密钥层次

```
用户自定义密钥 (≥8位)            主密码 (≥4位)
      │                              │
      ▼                              ▼
PBKDF2(keySalt, 10万轮)     PBKDF2(mpSalt, 60万轮)
      │                              │
      ▼                              ▼
    REK (32B)    ◄──────────   KEK 加密/解密 REK
      │
      ▼
AES-256-GCM 加密/解密 .pvault 密码库
```

- **REK**：真正的加密密钥，32 字节
- **KEK**：从主密码派生，用于加密/解密 REK
- 恢复密钥直接派生 REK（验证：SHA-256(REK) 比对）

### 4.2 文件格式

```
.pvault 文件结构：

┌──────────────────────────────────────┐
│  Header (明文 JSON)                  │
│  {                                   │
│    "version": 2,                     │
│    "masterPasswordSalt": "<b64>",    │
│    "keySalt": "<b64>",               │
│    "encryptedRek": "<b64>",          │ 密文
│    "rekIv": "<b64>",                 │
│    "rekAuthTag": "<b64>",            │
│    "rekHash": "<b64>",              │ SHA-256(REK)
│    "payloadIv": "<b64>",             │
│    "payloadAuthTag": "<b64>"         │
│  }                                   │
├──────────────────────────────────────┤
│  Payload (二进制，AES-GCM 加密)      │
│  { vaults: [...], entries: [...],    │
│    trash: [...], createdAt, updatedAt │
│  }                                   │
└──────────────────────────────────────┘
```

### 4.3 密钥规则

- 随机生成：32 字节 hex → 64 位字符
- 自定义：≥8 位任意字符，PBKDF2(10万轮) 拉伸
- 首次全量展示一次，之后永远显示 `a1b2****x7y8`
- 密钥复制不走剪贴板定时清除

### 4.4 关键场景

| 场景 | 流程 |
|------|------|
| 正常 | 锁屏 → 主密码 → 解锁 |
| 忘记密钥 | 主密码解锁 → 导出明文 → 重新生成密钥 → 重新加密 |
| 忘记主密码 | 锁屏→重置数据→设新主密码→用旧密钥加载.pvault |
| 全丢 | 死档 |

## 5. 数据模型

### 5.1 VaultPayload（加密）

```js
{
  vaults: [
    { id: 1, name: "默认", idPrefix: 10000, nextId: 10001, createdAt: "ISO" }
  ],
  entries: [
    {
      id: 10001,                    // 全局唯一，前缀标识创建时所在 Vault
      website: "github.com",
      alias: "GitHub",
      account: "user@example.com",
      password: "plaintext...",
      description: "代码托管平台",   // 备注
      vaultIds: [1, 2],             // 所属 Vault 列表（支持多归属）
      visible: true,                // 主界面是否显示
      order: 0,                     // 拖拽排序
      createdAt: "ISO",
      updatedAt: "ISO"
    }
  ],
  trash: [                          // 回收站
    { entry: {...}, deletedAt: "ISO" }
  ],
  createdAt: "ISO",
  updatedAt: "ISO"
}
```

## 7. 功能详细说明

### 7.1 搜索
- Toggle 多选：网站/别称/账号自由组合，密码互斥
- 字符串模糊包含，不区分大小写，空格分隔 AND 逻辑
- 同等匹配时按 order 排序
- 全局复选框：跨所有 Vault 搜索

### 7.2 多 Vault
- 条目标签化，一个条目可属多个 Vault
- 主界面按 Vault 分组显示，分组头 + 条目
- Vault 按创建时间排序，支持设置页拖拽自定义顺序
- 删除 Vault：弹窗选"全部删除"或"移到第一个 Vault"
- ID 前缀：Vault 1 = 10000-19999, Vault 2 = 20000-29999...

### 7.3 回收站
- 删除的条目先进回收站（保留完整数据 + 删除时间）
- 设置页可逐条恢复/永久删除
- 托盘菜单可一键清空
- 回收站内容不导出
- 删除条目需二次确认

### 7.4 自动锁屏
- 选项：5/15/30分钟, 1/2小时, 永不
- 选"永不"→红色警告弹窗
- 即将锁屏前 30 秒状态栏闪烁

### 7.5 登录保护
- 默认 5 次错误后冷却 30 秒
- 设置可选 3/5/7/11/自定义(最高99)
- 冷却期间输入框禁用 + 倒计时

### 7.6 导入/导出
- **导出明文**：JSON，包含描述，不含 vaultIds
- **导出加密**：直接复制 .pvault 文件
- **导入明文**：JSON → 合并到当前 Vault，可多选归属
- **导入加密**：选 .pvault → 输原主密码 → 合并
- **冲突**：网站+别称+账号全匹配=冲突，逐条弹窗确认

### 7.7 文件安全
- **原子写入**：先写 `.pvault.tmp`，成功后再 `rename` 替换
- **自动备份**：保存前备份上一版为 `.pvault.bak`

### 7.8 窗口
- 默认 900×620，最小 700×450
- 关闭弹窗：托盘/退出 + 记住选择
- 系统原生窗口控件（最大化/最小化/关闭）

### 7.9 Toast 通知
- 复制成功/失败、保存成功、同步完成等操作显示短暂提示
